## Why MDX?

MDX lets you write JSX directly inside Markdown. This means you get the best of both worlds: the simplicity of Markdown for content, and the full power of React components when you need interactivity.

### Key Benefits

- **Component embedding** — drop interactive React components right into your posts
- **Syntax highlighting** — powered by `rehype-pretty-code` with beautiful themes
- **Frontmatter** — structured metadata for each post using `gray-matter`
- **Reading time** — auto-calculated based on content length

## The Architecture

The blog follows the Next.js App Router convention:

```
content/blog/          # MDX files with frontmatter
app/blog/              # Blog routes
  page.tsx             # Post listing with tag filters
  [slug]/page.tsx      # Individual post pages
lib/blog.ts            # Post parsing utilities
components/blog/       # MDX component overrides
```

## Code Example

Here is a simple utility for parsing frontmatter:

```typescript
import matter from 'gray-matter';

const { data, content } = matter(source);
console.log(data.title); // Post title
console.log(content);    // MDX content body
```
